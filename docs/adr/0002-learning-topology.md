# ADR-0002: Топология обучения (автономность самообучения)

**Статус:** Accepted
**Дата:** 8 июня 2026 г.
**Контекст:** Киллер-фича продукта — «автономность самообучения контура».

---

## Контекст

Главная ценность ERA XDR — самообучение. Наивная трактовка («каждый из 150 000
агентов самообучается») нереализуема: убивает требование «лёгкий клиент», создаёт
рассогласованность и неуправляемость. Нужна топология, которая даёт автономность
контуру **целиком**, оставляя агент лёгким и детерминированным.

## Решение: Hub-and-Spoke (концентратор и спицы)

| Уровень | Что делает | Что НЕ делает |
|---|---|---|
| **Клиент (Rust-агент)** | Детерминированная Sigma-детекция + лёгкий поведенческий baseline хоста (онлайн-статистика). Автономный response офлайн. **Самоадаптируется.** | НЕ обучает тяжёлые ML-модели. НЕ хранит историю. НЕ принимает глобальных решений. |
| **Сервер (AI Core)** | **Единственный самообучающийся мозг.** Учится на агрегированной телеметрии всего контура, дообучает детекторы, маппит на MITRE через LLM. | — |

**Различие в терминах:** агент **самоадаптируется** (подстраивает локальный
baseline), **самообучается только AI Core**.

**Почему интеллект в ядре:** APT в среднем находится в сети 200+ дней и проявляется
в **корреляции между хостами** (endpoint → identity → email). Один агент этой
картины не видит — её видит только централизованный мозг.

## Цикл обратной связи («стадный иммунитет», herd immunity)

```
Агент A ловит новую аномалию (локальный baseline сработал)
   → шлёт в AI Core: Envelope + локальный вердикт + detection.confidence
   → AI Core анализирует, подтверждает, генерирует новое Sigma-правило / веса
   → раздаёт обновлённый детектор ВСЕМ 150k агентам
   → все защищены от того, что увидел только агент A
```

## Фазы эволюции обучения

| Фаза | Что | Передаётся снизу вверх | Статус |
|---|---|---|---|
| **2** | Централизованное обучение внутри организации | Сырьё (Envelope) + локальный вердикт + confidence | Базовая поставка |
| **3** | **Federated learning внутри организации** | Градиенты/веса локальных моделей (не сырьё) | **Опция (upsell)** |
| **4** | **Межорганизационный / национальный обмен** | Только обезличенные знания (IoC, Sigma, TTP) | **Опция (upsell)** |

> **Бизнес-правило (зафиксировано с заказчиком стратегии):** федеративные функции
> Фаз 3 и 4 предлагаются клиентам **как платная опция, НЕ по умолчанию.**
> Базовая поставка работает полностью автономно, без обмена данными наружу контура.

### Фаза 3 — Federated learning (опция)

Агенты шлют не сырьё, а **градиенты/веса** локальных моделей; AI Core их агрегирует.
Снижает внутренний трафик и усиливает privacy. Точка входа уже заложена —
поле `detection.confidence` в Envelope (см. ADR-0001).

### Фаза 4 — Национальный иммунитет (опция)

Двухуровневый hub-and-spoke между организациями:

```
Орг. A (банк)        Орг. B (госструктура)      Орг. C (холдинг)
 ERA XDR (air-gap)     ERA XDR (air-gap)          ERA XDR (air-gap)
      └──────────── только обезличенные знания ────────┘
                            │
              Национальный хаб (CERT / SOC уровня страны)
              агрегирует → общий иммунитет → раздаёт детекторы всем
```

**Незыблемый принцип границы данных** — сырьё и PII НИКОГДА не покидают контур
заказчика:

| Можно шарить вверх (в нацхаб) | Нельзя шарить никогда |
|---|---|
| IoC (хеши, IP, домены C2) | Сырая телеметрия |
| Sigma-правила новых атак | PII / данные сотрудников |
| TTP-паттерны (MITRE) | Внутренняя топология сети |
| Модельные градиенты (federated) | Содержимое файлов/почты |

**Стандарт обмена:** STIX/TAXII. Нацхаб = TAXII-сервер; каждая ERA XDR =
TAXII-клиент с approval-workflow и санитайзингом перед публикацией.

**Оговорка:** межорг-обмен — это governance/юридика ПРЕЖДЕ кода (NDA, политика
классификации, approval-цепочки). Технически закладываем (TAXII-совместимость,
чистые границы шаринга), включается при наличии правовой рамки.

## Последствия

**Плюсы:** лёгкий клиент сохранён; управляемость и согласованность детекторов;
ценность уровня национальной безопасности как апсейл; чёткие границы privacy.

**Минусы / обязательства:** AI Core — критичный компонент (нужна отказоустойчивость);
федеративные фазы требуют отдельной юридической и governance-проработки.
